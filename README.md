## 本地启动

### 1. 启动 Node 后端

```bash
cd ..\quiz-backend
npm install
npm run dev
```

后端默认监听 `http://127.0.0.1:3000`。
默认会从同级目录 `miniprogram-1\demos` 读取示例 PDF。

### 2. 启动微信小程序

用微信开发者工具打开项目根目录 `miniprogram-1`。

本地联调时，需要在开发者工具里关闭合法域名校验，否则小程序不能访问 `127.0.0.1`。

### 3. 本地题库

- 后端首次启动会自动导入同级目录 `miniprogram-1\demos` 下的 PDF。
- 小程序首页会从后端拉取题库和默认题量。
- 小程序端只负责刷题、提交结果和展示经纪人二维码。
- 题库管理和经纪人管理已经迁移到服务端后台。

### 4. 服务端后台

- 浏览器打开 `http://127.0.0.1:3000/admin/login`
- 默认管理员账号是 `admin`
- 默认管理员密码是 `admin`
- 题库上传、题干编辑、经纪人 ID 与二维码维护都在服务端后台完成

### 5. 小程序登录与分享归因

- 小程序启动时会调用 `wx.login`，再由后端通过 `code2Session` 换取 `openid`。
- 分享路径会自动附带当前应归属的中介人内部编号 `broker`。
- 如果当前用户绑定了中介人，则优先分享自己的内部编号。
- 如果当前用户不是中介人，则优先使用本地保存的首次来源内部编号。
- 如果本地和当前用户都无法命中经纪人，则回退到后台配置的默认经纪人。
